# 设计模式

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

设计模式是经过验证的、可复用的软件设计解决方案，分为创建型、结构型和行为型三大类，用于解决特定场景下的常见设计问题。

## 上下文

Java 开发中真正高频出现的设计模式包括：单例（全局唯一实例）、工厂（Map+Supplier 解耦对象创建）、策略（接口+多实现消除 if-else）、观察者（Spring @EventListener 事件机制）、代理（Spring AOP 底层）、模板方法（抽象类定义骨架，子类实现步骤）、建造者（Lombok @Builder 链式构造）、适配器（SLF4J 统一日志接口）。设计模式不是用来"背"的，而是用来解决具体问题的——理解每种模式解决了什么问题、在 Spring 等框架中哪里用到了，比记住全部 23 种更有意义。
