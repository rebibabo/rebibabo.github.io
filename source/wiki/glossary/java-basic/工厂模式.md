# 工厂模式

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

工厂模式将对象创建逻辑封装在工厂方法中，调用方只需指定"要什么"而不需要知道"怎么创建"，实现创建与使用的解耦。

## 上下文

最实用的写法是用 `Map<String, Supplier<T>>` 替代 if-else 分支：将每种具体实现的构造函数引用存入 Map，根据 key 获取对应的 Supplier 并调用 `get()` 创建对象。新增类型只需往 Map 里加一项，不用改调用方代码。Spring 的 BeanFactory、JDBC 的 DriverManager、Calendar.getInstance() 都是工厂模式的实际应用。Spring Boot 的自动配置本质上就是工厂——根据 classpath 上的类和配置文件自动创建合适的 Bean。与策略模式的区别：工厂关注"创建哪个对象"，策略关注"执行哪种算法"，两者常配合使用。
