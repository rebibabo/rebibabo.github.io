# 新生代

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

新生代（Young Generation）是堆内存中用于存放新创建对象的区域，占堆的约 1/3，内部划分为 Eden 区和两个 Survivor 区（S0、S1），发生 Minor GC。

## 上下文

新生代采用**标记-复制算法**：新对象首先在 Eden 区分配；Eden 满了触发 Minor GC，存活对象复制到 Survivor 区（S0 或 S1），年龄标记为 1；之后每次 Minor GC 在 Eden 和当前 Survivor 之间进行，存活对象复制到另一个 Survivor 并年龄 +1，两个 Survivor 轮流交替使用，始终有一个为空。对象年龄达到默认阈值 15（`-XX:MaxTenuringThreshold`）后晋升到老年代。Eden:S0:S1 默认比例为 8:1:1（`-XX:SurvivorRatio=8`），Eden 占比大是因为大部分对象"朝生夕死"——一次 Minor GC 通常 95% 的对象都是垃圾，只需复制少量存活对象。Minor GC 频率高但每次很快。
