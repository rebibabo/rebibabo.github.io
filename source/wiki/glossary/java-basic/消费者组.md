# 消费者组

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

Kafka Consumer Group（消费者组）是一组共享同一个 `groupId` 的 Consumer 实例，组内以竞争消费方式分配分区（一个分区只给一个消费者），不同 Consumer Group 之间各自独立维护消费进度，可完整消费同一份消息流。

## 上下文

Consumer Group 的核心规则：组内竞争消费——分区数决定了组内最大并行度，消费者数超过分区数时多出的消费者闲置；组间广播效果——不同 `groupId` 视为完全独立的消费者，各自维护一份 Offset 进度，互不影响。`groupId` 本质上就是一个独立的"消费进度"标识，Kafka 通过内部 Topic `__consumer_offsets` 记录每个 `(groupId, partition) -> offset` 的映射。当组内消费者数量变化（新增或宕机），会触发 Rebalance——分区和消费者的对应关系重新分配，Rebalance 期间该组消费短暂停止。同一业务实体的消息通过相同的 key 路由到同一分区，保证局部有序。
