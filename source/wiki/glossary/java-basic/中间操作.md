# 中间操作

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

中间操作（Intermediate Operation）是 Stream 管道中返回新 Stream 的操作，采用惰性求值策略——调用时不会立即执行，只有终端操作触发后才会随管道一起运行。

## 上下文

常用中间操作包括：`filter`（条件过滤）、`map`（一对一转换，如提取字段或类型变换）、`flatMap`（一对多展开，将嵌套结构展平一层，如将多行文本拆分成单词流）、`distinct`（去重，基于 equals 判断）、`sorted`（排序，支持自然排序或自定义 Comparator）、`limit`（截取前 n 个）、`skip`（跳过前 n 个，与 limit 组合可实现分页）、`peek`（调试用，"偷看"每个元素但不改变它们）。`mapToInt`、`mapToLong`、`mapToDouble` 是基本类型特化版本，返回 IntStream 等避免装箱开销。多个中间操作可以链式组合，构建完整的数据处理流水线。
