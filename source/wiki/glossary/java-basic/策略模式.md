# 策略模式

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

策略模式定义一组算法，将每个算法封装为独立的实现类并让它们可互相替换，使算法的变化独立于使用算法的客户端。

## 上下文

核心是"接口 + 多实现"：定义一个策略接口，每种算法写一个实现类，上下文持有接口引用而不关心具体是哪种实现。在 Spring 中可以通过 `Map<String, StrategyInterface>` 自动注入所有策略实现（key 是 Bean 名称），从而彻底消除 if-else 分支——新增策略只需加一个 `@Component` 类，不用改调用方代码。与工厂模式的区别：工厂关注创建哪个对象，策略关注执行哪种算法。枚举也能做策略——每个枚举值实现接口方法，适合策略数量固定的场景。

## 相关术语

- [[wiki/glossary/java-basic/设计模式|设计模式]] — 行为型设计模式，定义算法族并可互相替换
- [[wiki/glossary/java-basic/工厂模式|工厂模式]] — 常与策略模式配合：工厂创建策略对象，策略执行具体算法
- [[wiki/glossary/java-basic/枚举|枚举]] — 固定数量策略的轻量级实现（每个枚举值实现接口方法）
