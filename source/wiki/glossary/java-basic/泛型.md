# 泛型

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

泛型（Generics）是 Java 5 引入的类型参数化机制，允许类、接口、方法在定义时使用类型参数（如 `<T>`、`<E>`、`<K, V>`），在编译期进行类型检查，消除强制类型转换。

## 上下文

Java 5 之前集合只能存 Object，取出来必须强转，类型错误到运行时才暴露。泛型把类型检查提前到编译期，避免了 `ClassCastException`。常见形式：泛型类（`class Box<T>`）、泛型方法（`public static <T> T firstOrNull(List<T>)`）、泛型接口（`Comparable<T>`）。类型参数命名约定：`T`（Type）、`E`（Element）、`K`（Key）、`V`（Value）、`R`（Return）。Java 泛型通过类型擦除实现——泛型信息只存在于编译期，编译后所有类型参数被擦除为原始类型（上界类型或 Object），运行时 JVM 完全不知道泛型类型。这一设计是为了向后兼容 Java 5 之前的非泛型代码。
