# 包装类型（Wrapper Classes）

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

每个原始类型都有对应的包装类（例如 `int` 对应 `Integer`、`double` 对应 `Double`），用于在需要对象的场合——泛型、集合、反射、序列化等——让基本类型能融入 Java 的面向对象体系。包装类是不可变的（均为 final 类）。

## 上下文

大多数整数和字符包装类型在 -128 ~ 127 范围内使用了缓存池（`Integer.valueOf()` 返回同一对象），超出范围则新建对象。这是 `==` 比较包装类型出现"127 为 true、128 为 false"现象的根本原因——比较包装类的值永远用 `.equals()`。

Float 和 Double 没有缓存。Boolean 只缓存了 `TRUE` 和 `FALSE` 两个实例。自动拆箱时如果包装对象为 null，会抛出 NullPointerException，这在从数据库/Mapping 取值的场景中非常常见。

## 相关术语

- [[wiki/glossary/java-basic/原始类型|原始类型]] — 包装类型对应的基本数据类型，存储在栈上，访问速度快于包装类型
- [[wiki/glossary/java-basic/自动装箱|自动装箱]] — 编译器自动在原始类型和包装类型间转换，底层调用 valueOf() 方法
- [[wiki/glossary/java-basic/String|String]] — 同样是不可变的 final 类，拥有缓存池机制，与包装类型设计理念相似
