# 红黑树

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

红黑树是一种自平衡二叉搜索树，通过节点着色和旋转维持近似平衡，增删查时间复杂度均为 O(log n)，在 Java 中用于 TreeMap/TreeSet 底层和 HashMap 链表过长时的树化结构。

## 上下文

Java 选择红黑树而非 AVL 树的原因是：AVL 树严格平衡（左右子树高度差 <= 1），查询效率略高，但插入/删除时为了维持严格平衡需要更频繁的旋转；红黑树近似平衡，插入/删除的旋转次数更少，增删查综合性能更好——适合读写均衡的通用场景。在 HashMap 中，当链表长度超过 8 且数组容量 >= 64 时链表转换为红黑树（阈值 8 基于泊松分布，概率约百万分之六，正常分布下几乎不会触发）；退化回链表的阈值是 6（8 和 6 之间留 7 作为缓冲区间，避免反复树化-退化造成的性能抖动）。与普通二叉搜索树相比，红黑树防止了在有序插入数据时退化为链表。
