# 构造方法（Constructor）

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

构造方法是与类名相同、没有返回值类型的特殊方法，在 `new` 创建对象时被自动调用，负责初始化对象的状态。`new` 操作做三件事：在堆上分配内存、调用构造方法初始化、返回对象引用。

## 上下文

如果类中没有定义任何构造方法，编译器会自动生成一个无参构造；但一旦显式定义了任何构造方法，默认无参构造就会消失——这是继承和框架使用中的常见坑（Spring 等框架往往依赖无参构造）。

构造方法之间可以通过 `this(...)` 相互调用，但必须放在第一行。在继承体系中，子类构造方法必须先调用父类构造（`super(...)`），如果父类没有无参构造则必须显式调用有参构造，否则编译报错。Java 不支持像 C++ 那样的初始化列表语法，但可以通过 `this(...)` 链式调用来模拟。

## 相关术语

- [[wiki/glossary/java-basic/继承|继承]] — 子类构造必须先调用父类构造 super()，且必须放在构造方法第一行
- [[wiki/glossary/java-basic/static|static]] — static 代码块在类加载时执行一次，构造方法在每次实例化时执行
- [[wiki/glossary/java-basic/抽象类|抽象类]] — 抽象类可以有构造方法供子类调用，但抽象类本身不能被直接实例化
