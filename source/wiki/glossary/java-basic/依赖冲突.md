# 依赖冲突

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

依赖冲突指项目中不同依赖引入了同一库的不同版本，导致 Classpath 上存在多个同名但版本不同的 jar，可能引发 NoSuchMethodError 或 ClassNotFoundException。

## 上下文

Maven 处理冲突采用"最短路径优先"（路径短的版本胜出）和"最先声明优先"（路径长度相同时先声明的胜出）策略。可通过 `<exclusions>` 标签排除特定传递依赖，或通过 `<dependencyManagement>` 统一锁定版本。Gradle 默认使用最高版本策略（最新版本胜出），可通过 `resolutionStrategy` 强制指定版本。排查工具：`mvn dependency:tree -Dincludes=groupId:artifactId` 和 `gradle dependencies --configuration compileClasspath`。常见坑点：排除了必要的依赖导致运行时类缺失、依赖仲裁选定的版本与实际需要的 API 不兼容。
