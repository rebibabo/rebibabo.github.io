# 扩容

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

HashMap 扩容（resize）是指当元素数量超过容量 x 负载因子（默认 0.75）时，数组容量翻倍并重新计算所有元素在新数组中的位置（rehash），以避免哈希冲突恶化导致查询性能下降。

## 上下文

扩容时容量从 N 变为 2N（如 16->32）。容量变了，`hash & (capacity-1)` 的结果也会变，所以所有元素都需要重新计算位置。JDK 8 的优化：因为容量始终是 2 的幂，新容量多用了 1 个 bit，每个元素的 hash 值在这个新 bit 上要么是 0 要么是 1——是 0 则留在原下标，是 1 则移动到"原下标 + oldCapacity"。这比 JDK 7 的"全量重新计算 hash"效率高很多。扩容是一次性 O(n) 操作（复制所有元素），但因为不是每次 put 都发生，均摊下来不影响整体的 O(1) 性能。JDK 7 扩容时链表使用头插法，多线程同时扩容可能导致链表形成环形引用进而死循环；JDK 8 改为尾插法解决了死循环问题，但多线程 put 仍可能导致数据丢失。
