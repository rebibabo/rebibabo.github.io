# 双亲委派

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

双亲委派是 JVM 类加载器的一种协作机制：当需要加载一个类时，当前类加载器不会自己先加载，而是将请求委派给父类加载器，逐层向上直到 Bootstrap ClassLoader，父加载器都无法加载时才由自己加载。

## 上下文

类加载器分为三层：Bootstrap ClassLoader（加载 `java.*` 核心类库，C++ 实现，Java 中显示为 null）、Extension ClassLoader（加载 `jre/lib/ext` 目录下的扩展类）、Application ClassLoader（加载 classpath 中用户代码和第三方依赖）。双亲委派的核心价值是**安全性**（防止用户自定义 `java.lang.String` 替换核心类）和**一致性**（保证同一个类在 JVM 中只被加载一次）。某些场景需要打破双亲委派：Tomcat 为每个 Web 应用使用独立 ClassLoader、SPI 机制中 Bootstrap 加载的接口需要反向查找 classpath 上的实现类、热部署通过替换 ClassLoader 实现。
