# 垃圾回收

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

垃圾回收（GC）是 JVM 自动回收堆中不再使用的对象内存的机制，通过可达性分析判断对象是否存活，释放不可达对象占用的内存空间。

## 上下文

JVM 使用**可达性分析**（而非引用计数法）判断对象是否可回收：从一组 GC Roots（栈中局部变量、static 变量、常量引用、synchronized 锁持有对象）出发，沿引用链查找，不可达的对象即为垃圾。Java 提供四种引用强度：强引用（宁可 OOM 也不回收）、软引用（内存不够时回收，适合缓存）、弱引用（下次 GC 一定回收，如 WeakHashMap）、虚引用（跟踪对象回收状态）。三种基本 GC 算法为标记-清除（产生碎片）、标记-复制（新生代使用，无碎片但浪费一半内存）、标记-整理（老年代使用，无碎片但移动开销大）。主流回收器：Serial（单线程）、Parallel（吞吐量优先）、G1（Java 9+ 默认，可预测停顿）、ZGC（Java 15+，停顿 < 1ms，适合大堆）。
