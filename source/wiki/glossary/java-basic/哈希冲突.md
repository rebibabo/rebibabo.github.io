# 哈希冲突

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

哈希冲突是指不同的 key 经过哈希计算后落到了 HashMap 的同一个桶（数组下标），HashMap 通过"链表 + 红黑树"（链地址法）来解决冲突。

## 上下文

HashMap 用 `hash & (capacity - 1)` 计算桶下标。即便 hash 分布良好，有限的数组长度也会导致不同 key 映射到同一位置。解决方式：桶内用链表串起冲突节点，沿链表逐个比较 key（先比 hashCode 再比 equals）；JDK 8 优化为当链表长度超过 8 且数组容量 >= 64 时，链表转为红黑树（O(n) -> O(log n)）。为了减少冲突，HashMap 的扰动函数 `h ^ (h >>> 16)` 将 hashCode 的高 16 位与低 16 位异或，让高位差异也能影响数组下标，使分布更均匀。数组长度始终为 2 的幂使得 `hash & (capacity-1)` 等价于取模运算但比取模快。
