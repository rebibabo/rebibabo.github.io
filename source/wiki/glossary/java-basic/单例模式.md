# 单例模式

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

单例模式确保一个类在系统中只有一个实例，并提供全局访问点，适用于数据库连接池、配置管理器、线程池等"全局只需要一个"的场景。

## 上下文

Java 中有四种主流实现方式：饿汉式（类加载时创建，天然线程安全，最简单）、双重检查锁（DCL，volatile + synchronized，延迟加载）、静态内部类（懒加载+线程安全+无锁，利用 JVM 类加载机制）、枚举（Effective Java 推荐，天然防反射和反序列化破坏）。Spring 中 `@Service` 等注解默认就是单例，通常不需要自己写。四种方式各有取舍：饿汉式简单但可能浪费内存，DCL 经典但写法复杂易错，静态内部类兼顾懒加载和简洁，枚举最安全但不够灵活。
