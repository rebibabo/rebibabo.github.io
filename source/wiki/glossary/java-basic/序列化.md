# 序列化

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

序列化是将 Java 对象转换为字节流的过程，使其可以持久化到文件、存入数据库或通过网络传输；反序列化则是将字节流还原为 Java 对象。类必须实现 `Serializable` 标记接口才能被序列化。

## 上下文

Java 原生序列化使用 `ObjectOutputStream.writeObject()` 和 `ObjectInputStream.readObject()`，但存在安全漏洞、性能差、跨语言不兼容等问题，实际项目中很少使用。`transient` 关键字标记的字段不参与序列化（如密码字段），`static` 字段属于类不属于对象也不会被序列化。`serialVersionUID` 是版本号，用于反序列化时校验类的结构是否兼容——字段变更但版本号不变则取默认值，版本号改变则直接抛 `InvalidClassException`。实际项目中的替代方案：Jackson/Gson（JSON 格式，人类可读、跨语言，用于 REST API 和配置文件）、Protobuf（二进制格式，体积小速度快，用于 RPC 和微服务通信）、Kryo（Java 生态内最快，用于缓存和 Spark）。新项目直接用 Jackson，了解原生序列化机制主要用于面试和理解框架源码。
