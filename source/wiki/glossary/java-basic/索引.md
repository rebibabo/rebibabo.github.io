# 索引

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

数据库索引是用于加速数据查询的 B+ 树数据结构，类似于书的目录，通过维护索引页的有序存储避免全表扫描，以额外的写入和存储开销换取查询性能。

## 上下文

InnoDB 使用 B+ 树作为索引的默认数据结构——所有数据按主键（聚簇索引）的顺序存储，叶子节点包含完整行数据；非聚簇索引（二级索引）的叶子节点存储的是主键值，查询时需"回表"（通过主键再到聚簇索引中查找完整行）。覆盖索引指查询所需的所有列都包含在索引中，无需回表，性能最优。联合索引遵循最左前缀原则——只有查询条件从索引最左列开始且不跳过中间列时才能使用该索引。设计原则：为 WHERE、JOIN、ORDER BY 涉及的列建索引、避免过多索引拖慢写入、区分度低的列（如性别）不适合单独建索引。常见坑点：隐式类型转换导致索引失效（`WHERE varchar_col = 123`）、函数操作导致索引失效（`WHERE DATE(create_time) = '2024-01-01'`）、联合索引中间断档无法使用后续列。
