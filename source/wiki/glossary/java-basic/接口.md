# 接口（Interface）

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

接口定义一组行为的契约（"能做什么"），一个类可以实现多个接口以弥补 Java 单继承的限制。接口中的方法默认是 `public abstract`，字段默认是 `public static final` 常量。Java 8 引入了 `default` 方法（有实现体，实现类可继承或重写）和 `static` 方法（通过接口名调用），Java 9 引入了 `private` 方法（接口内部复用代码）。

## 上下文

default 方法解决了接口演进的历史难题——在 Java 8 之前，给接口加方法意味着所有实现类都必须修改。default 方法允许在接口中提供默认实现，已有实现类完全不受影响（如 `List.sort()`）。

只有一个抽象方法的接口叫函数式接口（标注 `@FunctionalInterface`），可以用 Lambda 表达式代替匿名内部类。Java 8 在 `java.util.function` 包中内置了四个最常用的函数式接口：`Supplier<T>`、`Consumer<T>`、`Function<T,R>`、`Predicate<T>`。抽象类 vs 接口的选择原则：需要共享状态或构造方法用抽象类，否则优先用接口（更灵活、支持多实现）。
