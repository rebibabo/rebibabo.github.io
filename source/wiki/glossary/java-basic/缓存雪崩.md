# 缓存雪崩

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

缓存雪崩指大量缓存在同一时间集中过期，或 Redis 集群整体宕机，导致所有请求瞬间涌入后端数据库，可能造成数据库过载甚至系统崩溃。

## 上下文

与缓存击穿（单个热点 key 过期）不同，缓存雪崩是大量 key 同时出现问题。应对策略分两层面：预防层面——为 key 的过期时间添加随机偏移量（如 `expire_time = base_time + random(0, 300)`），避免集中过期；高可用层面——Redis 采用主从 + 哨兵或 Cluster 模式部署，保证服务可用性。也有部分方案使用永不过期 + 异步更新策略（后台定时刷新缓存、读写时发现快过期则异步刷新）。降级方案：数据库压力过大时启用限流（Sentinel）、熔断，部分非核心功能直接降级返回默认值。常见坑点：随机偏移量设置过小仍可能集中过期、主从切换期间短暂不可用、缓存预热不充分导致冷启动大量请求穿透。
