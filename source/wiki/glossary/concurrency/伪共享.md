# 伪共享

[[wiki/glossary/concurrency/index|返回词汇表]]

## 定义

False Sharing 是并发中的隐蔽性能问题：多个线程修改的是不同变量，但这些变量落在了同一条 Cache Line 上。MESI 协议下，一个核心的修改会让其他核心的 Cache Line 失效，即使它们根本没有访问同一个变量。

## 上下文

表面看两个线程各更新各的变量互不影响，但缓存层面它们在争抢同一条 Cache Line——这被称为"伪"共享，因为不是共享变量本身。典型解决方案：在变量之间插入填充字节（padding），确保真正会被并发修改的变量落在不同的 Cache Line 上。

JDK 中的体现：`LongAdder` 的 Cell 用 `@Contended` 注解隔离；`ConcurrentHashMap` 的计数器 CounterCell 同样需要缓存行隔离。JDK 8 提供了 `@sun.misc.Contended` 注解（JDK 9+ 移到 `jdk.internal.vm.annotation.Contended` 并需 JVM 参数 `-XX:-RestrictContended` 开启）。
