# Memory Barrier（内存屏障 / 内存栅栏）

[[wiki/glossary/concurrency/index|返回词汇表]]

## 定义

Memory Barrier 是 CPU 层面的特殊指令，强制在屏障两侧的指令之间建立顺序约束：屏障前的内存操作必须先完成，屏障后的内存操作必须后执行。它是 JMM 在硬件层的实现基础。

## 上下文

内存屏障主要分两类：**Load Barrier**（读屏障）强制 CPU 从缓存/内存重新读取，而不用过期的本地缓存值；**Store Barrier**（写屏障）强制 CPU 将 Store Buffer 中的数据刷新到缓存，让其他核心可见。

JMM 中的 `volatile` 读写、`synchronized` 的进入/退出、`Lock.lock()/unlock()`，在底层都会插入相应的内存屏障。理解屏障不是为了手写汇编，而是为了理解为什么这些同步机制能可靠工作。
