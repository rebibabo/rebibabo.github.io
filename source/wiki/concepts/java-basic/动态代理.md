# 动态代理

[[wiki/concepts/java-basic/注解与反射|返回注解与反射]]

## 这一层回答什么问题

> 动态代理是什么？JDK 和 CGLIB 代理有什么区别？为什么 `@Transactional` 内部调用 this 会失效？

动态代理是 AOP 的底层机制。理解它，`@Transactional`、`@Cacheable`、Spring AOP 的限制就一目了然。

## 静态代理 vs 动态代理

静态代理：为每个接口手写一个代理类。一个接口一个代理，类爆炸。

动态代理：运行时动态生成代理类。一个 `InvocationHandler` 可以为所有接口工作。

## JDK 动态代理

```java
InvocationHandler handler = (proxy, method, args) -> {
    System.out.println("log before");
    Object result = method.invoke(target, args);  // 调真实对象
    System.out.println("log after");
    return result;
};

// 运行时生成代理类，实现 MyService 接口
MyService proxy = (MyService) Proxy.newProxyInstance(
    MyService.class.getClassLoader(),
    new Class[]{MyService.class},
    handler
);
```

**前提**：目标类必须实现接口。生成的代理类 `$Proxy0` 实现了你指定的接口，所有方法调用先经过 `InvocationHandler`。

## CGLIB 代理

当目标类没有实现接口时，Spring 用 CGLIB——通过继承目标类生成子类代理：

```java
// CGLIB 生成的代理类是目标类的子类
// UserService$$EnhancerBySpringCGLIB extends UserService
```

**限制**：目标类和方法不能是 `final` 的（`final` 类不可继承、`final` 方法不可重写）。

## 为什么内部调用 this 会失效

```java
@Service
public class UserService {
    public void methodA() {
        this.methodB();  // ❌ this 是原始对象，不是代理
    }
    
    @Transactional
    public void methodB() { ... }
}
```

代理模式的本质：外部调用者拿到的是代理对象。但 `this` 始终指向原始对象——`this.methodB()` 绕过了代理，AOP 切面不生效。

解决方案：注入 `self`（`@Autowired private UserService self; self.methodB()`），或拆到不同类。
