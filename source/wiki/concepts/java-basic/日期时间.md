# 日期时间

[[wiki/concepts/java-basic/数据类型|返回数据类型]]

## 这一层回答什么问题

> `java.util.Date` 为什么不推荐用了？`LocalDateTime` 和 `ZonedDateTime` 有什么区别？跨时区怎么处理？

Java 8 的 `java.time` 是对旧 Date/Calendar API 的彻底翻新。核心设计理念：**不可变 + 线程安全 + 语义清晰**。

## 旧 API 的问题

`java.util.Date` 有三个致命问题：
- **可变**：多线程共享时线程不安全
- **语义混乱**：`Date` 同时表示日期、时间、时间戳，无法区分"我只想要日期"
- **API 丑陋**：`getYear()` 返回的是当前年份减 1900，`getMonth()` 从 0 开始

`SimpleDateFormat` 更是线程不安全的——多线程共用一个实例会导致结果错乱或异常。

## 核心类

```
LocalDate         ← 只有日期（2026-07-06）
LocalTime         ← 只有时间（14:30:00）
LocalDateTime     ← 日期 + 时间，不带时区
ZonedDateTime     ← 日期 + 时间 + 时区
Instant           ← 时间戳（UTC 1970-01-01T00:00:00Z 起纳秒数）
Duration          ← 时间间隔（时分秒纳秒）
Period            ← 日期间隔（年月日）
DateTimeFormatter ← 格式化和解析
```

所有核心类都是不可变的——每次操作返回新对象，不修改原对象。线程天然安全。

## 选择建议

| 场景 | 用什么 |
|------|--------|
| 生日、纪念日 | `LocalDate` |
| 每天的闹钟 | `LocalTime` |
| 本地事件（不关心时区） | `LocalDateTime` |
| 需要跨时区、存储到 DB | `Instant` 存，展示时转 `ZonedDateTime` |
| 向用户展示时间 | `ZonedDateTime`，始终保存时区信息 |

**关键原则**：数据库存 UTC 时间戳（`Instant`），展示时根据用户时区转换（`ZonedDateTime`）。不要存带时区的字符串。

## DateTimeFormatter

替代 `SimpleDateFormat`，线程安全：

```java
DateTimeFormatter fmt = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");
String str = LocalDateTime.now().format(fmt);
LocalDateTime dt = LocalDateTime.parse("2026-07-06 14:30:00", fmt);
```
