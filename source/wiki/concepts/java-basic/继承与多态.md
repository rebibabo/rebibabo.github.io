# 继承与多态

[[wiki/concepts/java-basic/面向对象|返回面向对象]]

## 这一层回答什么问题

> 为什么要继承？多态是怎么工作的？重写和重载有什么区别？`Object` 的三个核心方法为什么要一起看？

继承和多态是 OOP 最核心的机制。继承解决代码复用，多态解决代码扩展——两个合在一起，才让 Java 的面向对象真正有工程价值。

## 继承（extends）

Java 是**单继承**——一个类只能有一个直接父类。但这不代表表达能力受限：通过接口的多实现可以弥补。

**关键规则**：
- 子类构造方法的第一行必须调父类构造方法。如果不显式写 `super(xxx)`，编译器自动插入 `super()`。如果父类没有无参构造方法，你必须显式调 `super` 并传参数
- 父类先初始化：父类构造方法执行完后，子类构造方法才继续
- 所有类的最终祖先都是 `Object`

## 方法重写（Override）vs 重载（Overload）

| | Override | Overload |
|------|----------|----------|
| 发生位置 | 父子类之间 | 同一个类中 |
| 方法签名 | 必须完全相同 | 参数列表必须不同 |
| 返回值 | 相同或协变（子类型） | 可以不同 |
| 访问权限 | 不能比父类更严 | 无限制 |
| 绑定时机 | 运行时（动态绑定） | 编译时（静态绑定） |

## 多态：为什么重要

```java
Animal a = new Dog();
a.makeSound();  // 调 Dog 的，不调 Animal 的
```

核心是**动态绑定**——调用哪个方法由运行时对象的实际类型决定，不由引用类型决定。

多态让代码"对接口编程"成为可能：一个方法接收 `List` 参数，传 `ArrayList` 或 `LinkedList` 都能跑。Spring 的 DI 就是多态的工程化应用——你写 `@Autowired EmailService es`，容器注入的是 `EmailServiceImpl`。

## Object 三件套

`toString()`、`equals(Object)`、`hashCode()` 是每个对象都继承的三个方法。

**equals 和 hashCode 的契约**：两个对象 equals 相等 → 它们的 hashCode 必须相等。反过来不成立（哈希冲突）。

为什么？HashMap 先用 hashCode 定位桶，再用 equals 在桶内匹配。如果 equals 相等的两个对象 hashCode 不同，它们会落到不同桶里，HashMap 找不到。

**重写 equals 就必须重写 hashCode**——这是所有 Java 开发者都要记住的规则。
