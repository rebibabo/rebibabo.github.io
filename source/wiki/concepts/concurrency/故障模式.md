# 故障模式与安全停止

[[wiki/concepts/concurrency/并发总图|返回并发总图]]

## 定义

这一页关注的不是“正常情况下怎么跑”，而是：

> 并发系统会怎么出错，以及怎样让它可诊断、可恢复、可停止。

## 这一层包含什么

- 死锁
- 活锁
- 线程饥饿
- 中断与安全停止
- 线程池里任务取消和资源回收

## 为什么它是横切主题

这些问题不会只出现在某一个 API 上。

它们横跨：

- 执行模型
- 锁与同步
- 异步任务调度
- 任务系统设计

所以单独拉一页更适合后续持续补充。
