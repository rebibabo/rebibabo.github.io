# 元注解

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

元注解是用于修饰其他注解的注解，Java 提供四个核心元注解来定义自定义注解的行为：`@Target`、`@Retention`、`@Documented`、`@Inherited`。

## 上下文

`@Target` 指定注解可以贴在哪些元素上，常用值包括 TYPE（类/接口）、METHOD（方法）、FIELD（字段）、PARAMETER（参数）、CONSTRUCTOR（构造方法）等。`@Retention` 控制注解保留到哪个阶段：SOURCE 只在源码中（如 `@Override`，编译后丢弃）、CLASS 保留到 class 文件但运行时不可读（默认值）、RUNTIME 保留到运行时可通过反射读取（框架注解必须用这个）。`@Documented` 控制注解是否出现在 Javadoc 中。`@Inherited` 控制子类是否继承父类的注解。Java 8 增加了 `@Repeatable`，允许同一个注解在同一位置重复使用。理解 Retention 三级别是使用注解的关键——SOURCE 给编译器看，RUNTIME 给反射看，CLASS 很少用到。
