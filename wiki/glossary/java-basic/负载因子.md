# 负载因子

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

HashMap 的负载因子（Load Factor）决定了数组使用到什么程度就触发扩容，默认值 0.75，扩容阈值 = 容量 x 负载因子。

## 上下文

负载因子是时间与空间的折中：接近 1.0 时空间利用率高，但哈希冲突概率增大，桶内链表变长，查询变慢（省空间但费时间）；接近 0.5 时冲突概率低，但数组经常半空，浪费内存（费空间但省时间）。0.75 是大量统计实验得出的经验值，在大多数场景下是时间和空间的最佳平衡点。以默认容量 16 为例，阈值 = 16 x 0.75 = 12，即放入第 13 个元素时触发扩容。扩容时容量翻倍（16->32），所有元素需要 rehash 重新计算在新数组中的位置——JDK 8 优化后每个元素的新位置只有两种可能：原下标，或原下标+旧容量。
