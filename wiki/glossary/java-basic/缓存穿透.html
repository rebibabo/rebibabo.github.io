缓存穿透

[[wiki/glossary/java-basic/index|返回词汇表]]

定义

缓存穿透指查询一个数据库中根本不存在的数据，由于缓存中也没有该 key，每次请求都会穿透缓存直接打到数据库，造成数据库压力剧增。

上下文

攻击者可能利用缓存穿透构造大量不存在的 key（如 id=-1）进行恶意攻击。解决方案有三种：（1）缓存空值——即使数据库查不到也缓存一个 null 或空对象，设置较短的过期时间防止占用过多内存；（2）布隆过滤器——用极小内存空间判断一个 key 是否可能存在，说不存在则一定不存在，说存在可能误判。适合 key 空间有限且集中的场景；（3）增强参数校验——在业务层拦截明显非法的输入（负数 ID、超长字符串等），从源头减少无效查询。常见坑点：缓存空值方案要考虑空值过多导致 Redis 内存膨胀，布隆过滤器不支持删除元素（可用计数型布隆过滤器但实现复杂）。
