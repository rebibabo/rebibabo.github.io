# 类型擦除

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

类型擦除（Type Erasure）是 Java 泛型的实现机制：编译器在编译期检查类型安全后，将所有泛型类型信息擦除并替换为原始类型（上界类型或 Object），同时在必要处自动插入强制类型转换，运行时 JVM 完全感知不到泛型。

## 上下文

类型擦除带来五个核心后果：(1) 运行时无法获取泛型类型——`List<String>` 和 `List<Integer>` 的 `getClass()` 都是 `ArrayList`；(2) 不能用基本类型作为类型参数，只能用包装类型（`List<Integer>` 而非 `List<int>`）；(3) 不能创建泛型数组 `new T[10]`；(4) 不能用 `instanceof` 检查泛型类型（只能用 `instanceof List<?>`）；(5) 不能 `new T()`。Java 选择类型擦除而非 C++ 那样为每种类型生成独立代码，是为保证向后兼容——Java 5 之前数百万行非泛型代码无需重编译即可继续运行。代价是丢失运行时类型信息。
