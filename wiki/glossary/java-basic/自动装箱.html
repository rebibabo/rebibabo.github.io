自动装箱与自动拆箱（Autoboxing & Unboxing）

[[wiki/glossary/java-basic/index|返回词汇表]]

定义

自动装箱是 Java 5 引入的编译器特性，自动将原始类型转换为对应的包装类型（如 Integer a = 42 编译后等价于 Integer.valueOf(42)）。自动拆箱则相反，自动将包装类型转换为原始类型（如 int b = a 编译后等价于 a.intValue()）。

上下文

自动装箱让原始类型和包装类型可以无缝混用，尤其是在集合操作中。但语法糖掩盖了两个关键陷阱：一是缓存池导致的 == 假象（-128~127 内装箱返回同一对象，之外返回新对象）；二是自动拆箱时的 NPE 风险——如果包装对象为 null，调用 intValue() 直接 NullPointerException。

自动装箱发生在赋值、方法参数传递、算术运算等场景。由于每次装箱都可能创建新对象，在热点循环中大量装箱会产生 GC 压力，应从性能角度关注。

相关术语

  • [[wiki/glossary/java-basic/原始类型|原始类型]] — 装箱的源类型，自动装箱将原始类型值包装为对应的包装类型对象
  • [[wiki/glossary/java-basic/包装类型|包装类型]] — 装箱的目标类型，自动装箱底层调用包装类的 valueOf() 方法
