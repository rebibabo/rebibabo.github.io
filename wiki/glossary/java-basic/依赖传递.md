# 依赖传递

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

依赖传递是构建工具自动引入一个依赖项所需的其他间接依赖（传递性依赖）的机制，避免开发者手动声明所有嵌套依赖。

## 上下文

在 Maven 中，当项目声明了对库 A 的依赖，Maven 会自动解析并下载 A 所依赖的库 B、C，形成依赖树（`mvn dependency:tree` 可查看）。传递性受 scope 影响：compile 范围的依赖会传递，provided 和 test 不会传递。依赖冲突时 Maven 遵循"最短路径优先"和"最先声明优先"原则仲裁版本。Gradle 中 `api` 的依赖可传递、`implementation` 只在当前模块可见。常见坑点：传递引入了预期外的旧版本，导致运行时 MethodNotFoundError；传递依赖过多导致 fat jar 体积膨胀。
