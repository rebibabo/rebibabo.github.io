# 继承（Inheritance）

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

继承是面向对象的代码复用机制。子类通过 `extends` 继承父类，获得父类的所有非 private 成员（字段和方法）。Java 只支持单继承——一个类只能有一个直接父类，但所有类都隐式继承 `Object`，所以每个对象都有 `toString()`、`equals()`、`hashCode()` 等方法。

## 上下文

子类构造时必须先完成父类的构造——如果不显式写 `super()`，编译器自动插入无参的 `super()` 调用，且必须放在子类构造方法的第一行。如果父类没有无参构造，子类必须显式调用父类的有参构造，否则编译报错。这是继承体系中常见的构造链陷阱。

`super` 关键字用于调用父类构造方法和被重写的方法。Java 的单继承限制是对 C++ 多继承菱形问题的回避方案——需要多继承效果时用接口。实际开发中更推崇"组合优于继承"原则，因为继承会带来强耦合。

## 相关术语

- [[wiki/glossary/java-basic/多态|多态]] — 继承是多态的前提，通过向上转型实现父类引用指向子类对象
- [[wiki/glossary/java-basic/方法重写|方法重写]] — 子类通过重写父类方法实现差异化行为，是继承的核心扩展机制
- [[wiki/glossary/java-basic/抽象类|抽象类]] — 作为父类模板，提供默认实现并强制子类重写抽象方法
- [[wiki/glossary/java-basic/接口|接口]] — 接口弥补 Java 单继承限制，一个类可以实现多个接口获得多继承效果
