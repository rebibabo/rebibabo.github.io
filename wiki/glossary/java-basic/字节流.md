# 字节流

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

字节流（InputStream/OutputStream）是 Java I/O 体系中处理二进制数据的抽象基类，以字节（8 位）为最小读写单位，适用于图片、音频、视频、压缩包等所有非文本数据的读写。

## 上下文

`InputStream` 的 `read()` 返回 int 类型的字节值（0-255），返回 -1 表示流结束；`OutputStream` 的 `write(int b)` 写入单个字节。对于文件操作常用 `FileInputStream` / `FileOutputStream`，实际开发中会配合 `BufferedInputStream` / `BufferedOutputStream` 装饰器添加缓冲区提升性能。字节流可以处理任何类型的数据（包括文本），但文本文件更推荐使用字符流以避免编码问题。传统字节流属于 BIO（同步阻塞 I/O），底层每次 read/write 都可能触发系统调用；现代 Java 对小文件的读写推荐使用 `Files.readAllBytes()` / `Files.write()` 等 NIO 工具方法。
