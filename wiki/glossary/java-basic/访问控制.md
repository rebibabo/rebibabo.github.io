# 访问控制（Access Modifiers）

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

Java 有四种访问修饰符，控制类、方法、字段在不同作用域中的可见性：`private`（仅本类）、默认/包访问（本类 + 同包）、`protected`（本类 + 同包 + 子类）、`public`（任何地方）。权限从严到宽依次递增。

## 上下文

面向对象的封装原则要求字段几乎总是 `private`，通过 getter/setter 提供受控的访问入口。对外暴露的 API 用 `public`，内部辅助方法用 `private`。`protected` 主要用于为子类预留扩展点——子类可以访问父类 protected 成员，但不同包的子类只能通过继承访问，不能通过父类实例访问。

默认（包级）访问在实际项目中较少刻意使用，更多是忘记写修饰符的结果。类本身也可以用 `public` 或默认修饰（一个 .java 文件最多一个 public 类）。

## 相关术语

- [[wiki/glossary/java-basic/构造方法|构造方法]] — 构造方法通常用 public，单例模式中常将构造方法设为 private 阻止外部实例化
- [[wiki/glossary/java-basic/static|static]] — static 成员通过类名访问，访问控制修饰符同样适用于 static 字段和方法
- [[wiki/glossary/java-basic/final|final]] — final 常与 private/public 搭配定义不可变常量或防止子类重写核心方法
