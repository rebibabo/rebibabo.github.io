动态代理

[[wiki/glossary/java-basic/index|返回词汇表]]

定义

动态代理是在运行时动态生成代理类，通过 InvocationHandler 拦截方法调用并在调用前后添加额外逻辑（如日志、事务、权限），核心 API 为 Proxy.newProxyInstance。

上下文

JDK 动态代理要求目标类必须实现接口，JVM 在运行时生成一个实现该接口的代理类（如 $Proxy0），代理类的每个方法只做一件事——将调用转发给 InvocationHandler.invoke()。Handler 内部通过反射调用真实对象的方法，并在前后添加额外逻辑。典型应用场景：Spring AOP（@Transactional 事务管理）、Spring Security（权限校验）、MyBatis Mapper（接口方法转 SQL）、RPC 框架（方法调用转网络请求）。JDK 代理 vs CGLIB 代理：JDK 代理基于接口，CGLIB 基于子类继承目标类（不需要接口），但不能代理 final 类和 final 方法。Spring Boot 2.0+ 默认使用 CGLIB。常见陷阱：同一个类内部调用 this.xxx() 不走代理，导致 @Transactional 等注解失效。
