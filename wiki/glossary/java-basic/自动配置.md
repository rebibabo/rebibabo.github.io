# 自动配置

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

自动配置（Auto-Configuration）是 Spring Boot 的核心机制，通过 `@EnableAutoConfiguration` 根据类路径上的 jar 依赖和已有 Bean 自动推断并配置应用所需的 Spring Bean。

## 上下文

自动配置的工作原理：`spring-boot-autoconfigure` 模块包含大量 `xxxAutoConfiguration` 类，每个类上标注 `@ConditionalOnClass`（类路径存在某类时生效）、`@ConditionalOnMissingBean`（用户未自定义 Bean 时生效）、`@ConditionalOnProperty`（配置属性满足条件时生效）等条件注解。例如 `DataSourceAutoConfiguration` 检测到 `DataSource` 类存在且用户未手动配置时，自动创建数据源。自动配置类通过 `spring.factories` 文件注册，Spring Boot 启动时逐个加载评估。排除不需要的自动配置：`@SpringBootApplication(exclude = DataSourceAutoConfiguration.class)` 或 `spring.autoconfigure.exclude` 配置。常见坑点：自动配置的行为与预期不符（往往是因为用户定义了某个 Bean 使 `@ConditionalOnMissingBean` 跳过了自动配置）、条件注解不满足时静默跳过导致"找不到 Bean"报错。
