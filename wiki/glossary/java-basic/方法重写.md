# 方法重写（Override）

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

方法重写是子类重新定义父类方法的行为。重写的方法名和参数列表必须相同，返回值类型可以相同或是子类（协变返回），访问修饰符不能比父类更严格（如父类 protected 可以重写为 public），不能抛出比父类更宽的受检异常，final 方法和 static 方法不能被重写。

## 上下文

重写和重载名字相似但本质完全不同：重写发生在父子类之间、运行时动态绑定；重载发生在同一个类中、编译时静态绑定（只看参数列表）。`@Override` 注解不是必须的，但强烈建议使用——编译器会检查是否真的重写了父类方法，防止拼错方法名变成新方法。

static 方法的同名定义叫"隐藏（Hiding）"，调用时看引用声明类型。子类调用父类被重写的方法用 `super.methodName()`，常见于在子类扩展逻辑中先执行父类的默认行为。重写是多态的实现基础——没有重写，多态就没有意义。

## 相关术语

- [[wiki/glossary/java-basic/继承|继承]] — 重写发生在父子类之间，是继承机制中最核心的行为扩展方式
- [[wiki/glossary/java-basic/多态|多态]] — 重写是多态的实现基础，运行时通过动态绑定决定调用子类重写的方法
- [[wiki/glossary/java-basic/final|final]] — final 方法禁止子类重写，用于保护父类核心行为不被篡改
