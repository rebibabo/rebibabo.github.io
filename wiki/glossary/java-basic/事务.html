事务

[[wiki/glossary/java-basic/index|返回词汇表]]

定义

数据库事务是一组不可分割的操作单元，遵循 ACID 特性（原子性、一致性、隔离性、持久性），要么全部成功提交，要么全部失败回滚，保证数据的一致性状态。

上下文

MySQL InnoDB 引擎默认隔离级别是 REPEATABLE_READ（可重复读），通过 MVCC（多版本并发控制）实现——每次读操作读取 Undo Log 中的快照版本，避免阻塞但代价是可能读到旧数据。四种隔离级别从低到高：READ_UNCOMMITTED（脏读）、READ_COMMITTED（不可重复读）、REPEATABLE_READ（幻读，InnoDB 通过 Next-Key Lock 消除）、SERIALIZABLE（性能差，基本不用）。Spring 中通过 @Transactional 声明事务边界，默认只对 RuntimeException 和 Error 回滚（检查异常不自动回滚）。常见坑点：同类内部方法调用导致 @Transactional 失效（AOP 代理问题）、事务内混用不同数据源导致事务不一致、大事务长时间占用锁和连接导致系统瓶颈。
