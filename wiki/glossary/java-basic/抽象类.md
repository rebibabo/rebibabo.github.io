# 抽象类（Abstract Class）

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

`abstract` 修饰的类为抽象类，不能被实例化。它可以包含抽象方法（只有声明没有实现，强制子类必须重写）和具体方法（提供子类可继承的默认实现），也可以有构造方法（供子类调用）和实例字段（为子类提供共享状态）。

## 上下文

抽象类设计表达"是什么"（is-a）的关系，适合在类层次结构中作为通用模板——向上抽取共同行为，向下强制差异化实现。一个类如果有任何抽象方法就必须声明为 abstract，但 abstract 类不一定有抽象方法（纯粹阻止实例化）。

与接口的核心区别在于：抽象类可以有构造方法和实例字段（维护共享状态），但只能单继承；接口可以多实现，Java 8+ 的 default 方法也提供了默认实现能力。现代 Java 的倾向是优先用接口，抽象类主要用于需要维护共享状态的场景。
