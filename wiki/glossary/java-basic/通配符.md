# 通配符

[[wiki/glossary/java-basic/index|返回词汇表]]

## 定义

通配符 `?` 是 Java 泛型中表示"某个未知类型"的占位符，分为无界通配符 `<?>`、上界通配符 `<? extends T>` 和下界通配符 `<? super T>` 三种，用于在使用泛型类型时放宽类型限制。

## 上下文

通配符与类型参数 `T` 的区别：`T` 是定义时声明的占位符（"这个类型我会在方法体内多次引用"），`?` 是使用时表示"我不关心具体类型"（通常作为方法参数）。`<?>` 适用于只读不关心类型的场景（如 `printList(List<?>)`）。`<? extends T>`（上界）只能读不能写——因为编译器不知道容器里具体是哪种 T 的子类，写入可能类型不安全。`<? super T>`（下界）只能写不能安全读——写入 T 一定是安全的但取出来的类型不确定。实际开发中通配符主要出现在方法参数中增加 API 的灵活性，遵循 PECS 原则判断该用哪种。
