# 上下文切换

[[wiki/glossary/concurrency/index|返回词汇表]]

## 定义

Context Switch 是操作系统从一个线程切换到另一个线程时，保存和恢复 CPU 寄存器、程序计数器、栈指针等执行上下文的过程。这是多线程并发的基础代价。

## 上下文

上下文切换本身有时间开销（保存/恢复寄存器、刷新 TLB 和缓存），但更重要的是间接代价：切换后新线程的 CPU Cache 几乎是冷的——之前的缓存数据需要重新加载，即"缓存预热"成本。

在 Java 并发中影响设计选择的点：为什么非公平锁吞吐更高（减少切换次数，让刚释放锁的线程更快重新获取）；为什么 ForkJoinPool 的 Worker 优先执行本地任务（保持 Cache 热度）；为什么 `LongAdder` 分散竞争比全局 CAS 好（减少因竞争导致的切换和缓存乒乓）。减少不必要的上下文切换是并发性能优化的核心目标之一。
