# 线程中断

[[wiki/glossary/concurrency/index|返回词汇表]]

## 定义

线程中断是 Java 中的协作式停止机制。外部线程调用 `interrupt()` 发出停止请求（设置中断标记），目标线程自己决定何时、如何响应该请求——它不是强制杀死线程。

## 上下文

中断机制的关键：`interrupt()` 设置中断标记 → 运行中任务通过 `isInterrupted()` 检查标记 → 阻塞中任务通过 `InterruptedException` 被唤醒。三种阻塞方式的响应差异：`lock()` 不响应中断；`lockInterruptibly()` 等锁期间被中断会抛出异常；`await()` 在条件等待期间被中断会抛出异常但要重新拿到锁后才抛。

常见的危险做法：catch 里吞掉中断（`catch(InterruptedException e) {}` 什么都不做）会导致停止请求丢失。正确做法是恢复中断标记 `Thread.currentThread().interrupt()` 再加退出逻辑。
