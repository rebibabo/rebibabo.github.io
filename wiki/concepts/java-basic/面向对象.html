面向对象

[[wiki/concepts/java-basic/Java-Basic-概念总图|返回概念总图]]

这一层回答什么问题

Java 里怎么用类来组织代码？继承、多态、接口这些机制到底在解决什么工程问题？

面向对象不是”背四大特性”——它是 Java 组织代码的基本方式。理解 OOP 不是为了应付面试，是为了看懂任何一个 Java 项目的类结构。

这层的主要分支

  • [[wiki/concepts/java-basic/类与对象|类与对象]] — 构造方法、this、访问控制、static、final
  • [[wiki/concepts/java-basic/继承与多态|继承与多态]] — extends、Override、动态绑定、Object 三件套
  • [[wiki/concepts/java-basic/接口与抽象|接口与抽象]] — 抽象类 vs 接口的选择、default 方法、函数式接口

类与对象

类是模板，对象是实例。new 关键字在堆上分配内存、调构造方法、返回引用。

  • 构造方法：初始化对象状态，可以用 this(xxx) 链式调用
  • 访问控制：private → default → protected → public 四级，控制可见性
  • static：属于类本身，所有实例共享。静态方法里没有 this
  • final：类不可继承 / 方法不可重写 / 变量不可修改

继承

extends 让子类复用父类的字段和方法。Java 是单继承，但可以通过接口多实现来弥补。

关键规则：

  • 子类构造必须先调父类构造（隐式 super()）
  • 所有类最终继承 Object
  • super.xxx() 调用父类方法

多态

同一方法调用在不同子类上表现出不同行为。核心机制是动态绑定——运行时根据对象实际类型决定调哪个版本。

多态的前提是：父类引用指向子类对象 + 子类重写了父类方法。

抽象类 vs 接口

两者都定义”子类必须做什么”，但使用场景不同：

维度 抽象类 接口
继承 单继承 多实现
可以有字段 是 只有常量
可以有构造方法 是 否
default 方法 是 Java 8+
适用 “是什么” “能做什么”

Object 三件套

toString() / equals() / hashCode() 是每个类都有的三个方法。关键契约：equals 相等的对象，hashCode 必须相等。重写 equals 不重写 hashCode，HashMap 里就找不到你的对象。

为什么 OOP 不只是”背概念”

单看定义，封装、继承、多态都很抽象。但放到真实场景里：

  • 封装 → 改内部实现不破坏外部调用方
  • 继承 → 复用代码，不重复写
  • 多态 → 一个方法接收接口类型，传什么实现都能跑
  • 抽象 → 定义规范，强制子类遵守

Spring 的 ApplicationContext 是一个接口，AnnotationConfigApplicationContext 是它的实现。你写 ApplicationContext ctx = ... 就是在用多态。
