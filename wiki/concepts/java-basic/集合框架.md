# 集合框架

[[wiki/concepts/java-basic/Java-Basic-概念总图|返回概念总图]]

## 这一层回答什么问题

> 数据多了怎么存？List、Set、Map、Queue 各有什么适用场景？HashMap 底层到底怎么工作的？

集合框架是 Java 中使用频率最高的 API 之一。不夸张地说，几乎所有 Java 程序都在用集合——但选错实现类、不理解底层原理，是性能问题的常见来源。

## 这层的主要分支

- [[wiki/concepts/java-basic/ArrayList与LinkedList|ArrayList 与 LinkedList]] — 动态数组 vs 双向链表、扩容机制、CPU 缓存视角
- [[wiki/concepts/java-basic/HashMap|HashMap]] — 数组+链表+红黑树、扰动函数、负载因子、扩容
- [[wiki/concepts/java-basic/ConcurrentHashMap|ConcurrentHashMap]] — JDK 7 分段锁 → JDK 8 CAS+synchronized 按桶加锁

## 集合框架全景

```
                    Iterable
                       ↑
                  Collection
                 ↙     ↓      ↘
              List    Set    Queue/Deque
              ↙  ↘    ↙  ↘      ↙  ↘
       ArrayList  HashSet  PriorityQueue
       LinkedList TreeSet  ArrayDeque
       Vector     LinkedHashSet

                    Map
                 ↙    ↓    ↘
           HashMap  TreeMap  Hashtable
           LinkedHashMap
           ConcurrentHashMap
```

两条线：`Collection`（存单个元素）和 `Map`（存键值对）。

## 核心选型

| 需求 | 选什么 | 为什么 |
|------|--------|--------|
| 需要索引、经常遍历 | `ArrayList` | O(1) 随机访问 |
| 需要不重复 | `HashSet` | O(1) 查重 |
| 需要 key-value | `HashMap` | O(1) 查找 |
| 需要排序 | `TreeSet` / `TreeMap` | 红黑树，O(log n) |
| 需要插入顺序 | `LinkedHashSet` / `LinkedHashMap` | 链表维护顺序 |
| 需要 FIFO 队列 | `ArrayDeque` | 比 LinkedList 快且省内存 |

## HashMap 底层：为什么这么设计

```
数组(桶) → 链表(Node) → 红黑树(TreeNode)
```

**put 流程：**
1. 计算 key 的 `hashCode()`，扰动函数 `(h = key.hashCode()) ^ (h >>> 16)` 让高位参与运算
2. `(n-1) & hash` 定位桶下标
3. 桶为空直接放；不为空遍历链表/树比较
4. 链表长度 ≥ 8 且数组长度 ≥ 64 → 转红黑树
5. 元素数 > 容量 × 0.75 → 扩容（容量翻倍）

**为什么容量是 2 的幂？** `(n-1) & hash` 等价于 `hash % n`，位运算更快。

**JDK 7 → 8 的关键变化：** 头插法 → 尾插法，解决了并发 resize 时的死循环问题。但 HashMap 仍然不是线程安全的——多线程场景用 `ConcurrentHashMap`。

## ArrayList 底层

动态数组，默认容量 10，扩容时增加到 1.5 倍，通过 `System.arraycopy` 复制。**均摊** O(1) 的 add——扩容不是每次发生。
