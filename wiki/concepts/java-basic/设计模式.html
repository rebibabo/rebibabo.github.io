设计模式

[[wiki/concepts/java-basic/工程实践|返回工程实践]]

这一层回答什么问题

设计模式是不是就是背 23 种？Java 生态里实际上最常用的有哪几个？Spring 里用了哪些？

设计模式不是在代码里生搬 UML——是在遇到某种重复出现的问题时，知道”这个问题已经有人想清楚了”。Java 生态中最常用的是单例、工厂、策略、代理、模板方法、建造者。

单例：全局唯一

四种实现：饿汉（启动即加载）、DCL（双重检查锁 + volatile）、静态内部类（类加载机制保证）、枚举（防反射/反序列化破坏，最安全）。

Spring Bean 默认 scope=singleton 就是单例的工程化应用。

工厂：封装对象创建

不直接 new，而是通过工厂。Spring 的 ApplicationContext.getBean() 就是超级工厂——你不用管对象怎么创建的，只要告诉容器”我需要这个”。

策略：运行时选算法

定义算法接口，各实现封装在独立类里。Spring 自动把接口的所有实现注入到 Map<String, Strategy>，天然消除 if-else。

代理：控制访问

动态代理是 Spring AOP 的底层。@Transactional 的方法调用实际是代理对象在前后加事务逻辑。

模板方法：骨架 + 填空

抽象类定义”第一步→第二步→第三步”的骨架，子类负责填空。JdbcTemplate 的 query() 就是——Spring 管理连接、关闭、异常，你只写 SQL 和映射。

建造者：分布构建

Lombok @Builder 是建造者模式的标杆应用。复杂对象构建不需要写一长串构造参数。
