# 枚举

[[wiki/concepts/java-basic/数据类型|返回数据类型]]

## 这一层回答什么问题

> `enum` 比 `static final` 常量好在哪？枚举为什么能实现单例和策略模式？`EnumSet` 为什么比 `HashSet` 快十倍？

枚举不只是一种"限定取值的常量"。它的本质是一个继承了 `java.lang.Enum` 的 final class，可以有构造方法、字段、方法，甚至可以实现接口。

## java

```java
public enum Day {
    MONDAY, TUESDAY, WEDNESDAY;
}
```

编译器把它变成类似这样：

```java
public final class Day extends Enum<Day> {
    public static final Day MONDAY = new Day("MONDAY", 0);
    public static final Day TUESDAY = new Day("TUESDAY", 1);
    public static final Day WEDNESDAY = new Day("WEDNESDAY", 2);
    private Day(String name, int ordinal) { super(name, ordinal); }
}
```

关键：每个枚举值是类的 `public static final` 实例，构造方法是 `private` 的，所以外界不能创建新实例。

## 为什么要用枚举替代 static final 常量

| | `static final int` | `enum` |
|------|------|------|
| 类型安全 | `setStatus(0)` — 0 是什么？ | `setStatus(Status.ACTIVE)` — 编译期类型检查 |
| 命名空间 | 全局，容易冲突 | 枚举内聚 |
| 可以带数据 | 不能 | 构造方法 + 字段 |
| 可以带行为 | 不能 | 方法 + 接口实现 |

## 带属性和方法的枚举

```java
public enum OrderStatus {
    PENDING(1, "待处理"),
    SHIPPED(2, "已发货"),
    DELIVERED(3, "已完成");

    private final int code;
    private final String desc;

    OrderStatus(int code, String desc) { this.code = code; this.desc = desc; }

    public static OrderStatus fromCode(int code) {  // 反查
        for (OrderStatus s : values()) if (s.code == code) return s;
        throw new IllegalArgumentException("unknown code: " + code);
    }
}
```

## 枚举实现策略模式

每个枚举值可以在 `{}` 里覆写方法，给出自己的实现：

```java
public enum Operation {
    PLUS  { double apply(double a, double b) { return a + b; } },
    MINUS { double apply(double a, double b) { return a - b; } };
    abstract double apply(double a, double b);
}
// 使用：Operation.PLUS.apply(3, 5) → 8
```

这比 if-else 或 switch 干净——新增运算只需加一个枚举值，不用改其他地方。

## 枚举实现单例

```java
public enum DatabaseConnection {
    INSTANCE;
    public void connect() { ... }
}
```

枚举单例天然防反射攻击（`Constructor.newInstance()` 对枚举抛异常）、防反序列化破坏（反序列化拿到的始终是同一个 INSTANCE）。这是 Effective Java 推荐的最佳单例实现方式。

## EnumSet 和 EnumMap

`EnumSet` 底层是位向量——一个 `long` 的 64 个 bit，每个 bit 表示一个枚举值是否在集合中。所有操作都是位运算，比 `HashSet` 快十倍。

`EnumMap` 底层是数组——枚举的 `ordinal()` 作为数组下标，O(1) 访问。
