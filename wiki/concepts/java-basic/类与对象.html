类与对象

[[wiki/concepts/java-basic/面向对象|返回面向对象]]

这一层回答什么问题

类怎么定义的？new 背后发生了什么？构造方法、this、访问控制、static、final 各管什么？

类与对象是 Java OOP 的起点。它关心的不是”为什么用 OOP”，而是”OOP 在 Java 里怎么写、每条规则有什么工程意义”。

类与对象的关系

类是模板，对象是实例化结果。new 做了三件事：在堆上分配内存 → 调构造方法初始化 → 返回引用。Java 里没有”对象变量”——只有”引用”，它存的是对象的地址。

构造方法

名称和类名相同、没有返回值。如果你不写，编译器自动生成一个无参构造方法；一旦你手动写了任何构造方法，默认的无参构造方法就消失了。

this(xxx) 可以在一个构造方法里调用另一个构造方法——但它必须是第一行语句。这样做是让所有构造方法最终收敛到同一个初始化入口。

this 关键字

三种用途：

  • this.field —— 区分成员变量和局部变量（当它们同名时）
  • this(xxx) —— 构造方法之间互相调用
  • return this —— 返回当前对象，实现链式调用（Fluent API）

访问控制

修饰符 类内 同包 子类 任何地方
private ✓
default ✓ ✓
protected ✓ ✓ ✓
public ✓ ✓ ✓ ✓

封装的核心：字段 private，通过 getter/setter 暴露——setter 里可以加校验逻辑。不要为了方便把所有字段改成 public。

static

static 表示”属于类，不属于实例”。所有实例共享同一份静态变量。静态方法没有 this，不能直接访问实例成员。

静态代码块 static { ... } 在类加载时执行一次，适合做静态资源初始化。

final

final 修饰 效果
变量 不能重新赋值（引用类型只锁引用）
方法 不能被子类重写
类 不能被继承

static final 组合是常量：属于类、不可修改。常量名约定用 UPPER_SNAKE_CASE。
