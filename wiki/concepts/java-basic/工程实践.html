工程实践

[[wiki/concepts/java-basic/Java-Basic-概念总图|返回概念总图]]

这一层回答什么问题

代码写完了，怎么保证它对？怎么管理版本？怎么部署上线？消息队列、设计模式、开发规范这些”非语言特性”在工程中扮演什么角色？

这一层关心的是”让代码从能跑变成能交付”。语言和框架教你写代码，工程实践教你写能上线、能维护、能协作的代码。

设计模式

不是”背 23 种模式”——而是理解 Java 生态中最常见的几种模式及其在框架中的实际应用：

模式 一句话 Spring 中的应用
单例 全局只有一个实例 Spring Bean 默认 singleton
工厂 封装对象创建 BeanFactory
策略 运行时切换算法 Map<String, Strategy> 自动注入
代理 控制对象访问 AOP、@Transactional
模板方法 定义骨架、子类填空 JdbcTemplate、HttpServlet
建造者 分步构建复杂对象 Lombok @Builder

单元测试

JUnit 5 是测试框架，Mockito 模拟依赖，MockMvc 测试 Controller 层。测试结构遵循 Given-When-Then。

测试不是”写完代码补几个用例”——它是设计工具。写测试的过程就是思考接口是否可测、依赖是否合理的过程。

Git

工作区 → 暂存区 → 本地仓库 → 远程仓库，四阶段模型。merge vs rebase 黄金原则：公共分支用 merge（保留历史），个人分支用 rebase（保持线性）。不要 rebase 已 push 的分支。

Tomcat 与 Servlet

一次 HTTP 请求的完整旅程：Connector 接收连接 → Filter 链 → DispatcherServlet → HandlerMapping → Interceptor 链 → Controller → 视图渲染 → Filter 链（反向）。

Filter 是 Servlet 规范层的，Interceptor 是 Spring MVC 层的。Filter 用于编码/CORS/XSS，Interceptor 用于鉴权/日志/性能统计。

消息队列

MQ 的三大核心价值：解耦（A 发消息，B 订阅，互不感知）、异步（非核心步骤异步处理）、削峰（MQ 缓冲，消费者匀速消费）。

维度 RabbitMQ Kafka
设计 消息代理（AMQP） 分布式日志
吞吐 万级/秒 百万级/秒
消费后 ACK 后删除 不删除，可重复消费
适用 业务消息 流数据、事件溯源

开发规范与工作流

代码规范（命名、格式、注释）、API 设计规范（RESTful、统一响应格式）、Git 分支模型（Git Flow/Trunk Based）、CI/CD 流水线、部署策略（灰度/蓝绿/金丝雀）——这些不是”加分项”，是团队协作的基础设施。

这一层和其他层的不同

前面 13 个概念都是在教”一个 Java 程序怎么写”。这一层教的是”一个 Java 项目怎么做”——它连接了代码和真实世界。
