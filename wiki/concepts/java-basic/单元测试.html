单元测试

[[wiki/concepts/java-basic/工程实践|返回工程实践]]

这一层回答什么问题

JUnit 5 怎么用？Mockito 模拟了什么？测试应该怎么写才不是”补用例”？

单元测试不是写完代码再补的——它是设计工具。写测试的过程就是在检验你的接口是否可测、依赖是否清晰。

JUnit 5

核心注解：@BeforeEach / @AfterEach（每个测试前后）、@BeforeAll / @AfterAll（整个测试类前后，方法必须是 static）。

Mockito：隔离依赖

@Mock 创建一个假对象，@InjectMocks 把 Mock 注入到被测试对象。when(xxx).thenReturn(yyy) 定义行为，verify(xxx, times(n)) 验证调用。

@Spy 是部分 mock——真实对象部分方法被 mock。不要滥用 @Spy，它暗示你的类需要重构。

三层测试

@WebMvcTest 只加载 Controller 层（MockMvc 模拟 HTTP），比 @SpringBootTest 快很多。@DataJpaTest 只加载数据层。
