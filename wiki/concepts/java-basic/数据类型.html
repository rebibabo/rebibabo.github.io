数据类型

这一层回答什么问题

Java 里数据怎么表示？基本类型和包装类型有什么区别？String 为什么特殊？枚举和日期也是类型吗？

“类型”不只是 int 和 String 的罗列——它是 Java 程序里一切数据的契约。类型选错了，轻则性能差，重则 bug 难查。

基本类型（8 种）

类别 类型 大小 默认值
整型 byte / short / int / long 1/2/4/8 字节 0
浮点 float / double 4/8 字节 0.0
字符 char 2 字节（UTF-16 码元）
布尔 boolean JVM 相关 false

整型字面量默认 int，浮点默认 double。long a = 100L、float b = 1.0f。

包装类型

每个基本类型对应一个包装类（int → Integer、long → Long 等）。核心点：

  • 自动装箱/拆箱：编译器在基本类型和包装类型之间自动转换
  • 缓存池：Integer.valueOf() 对 -128~127 做了缓存，== 比较时踩坑最多
  • NPE 风险：Integer a = null; int b = a; → NullPointerException

String

String 在 Java 里不是基本类型，但行为上像个”特殊类型”：

  • 不可变：底层是 final byte[]（Java 9+ Compact Strings），任何”修改”都产生新对象
  • 常量池：字面量 "hello" 优先从 String Pool 获取，不新建对象
  • StringBuilder：循环拼接用 StringBuilder 而不是 +，否则 O(n²) 对象创建

枚举

enum 不只是常量集合。它的本质是 final class extends Enum，可以有构造方法、字段和普通方法。枚举天然防反射、防反序列化，是推荐的单例实现方式。

日期时间

Java 8 的 java.time 是不可变 + 线程安全的日期 API：

  • LocalDate / LocalTime / LocalDateTime — 不带时区
  • ZonedDateTime — 带时区
  • Instant — 机器时间戳
  • DateTimeFormatter — 替代 SimpleDateFormat（线程不安全）

和其他概念的关系

类型系统是语言的核心基础设施。泛型在类型系统上加了一层”类型参数化”，集合框架用泛型来保证类型安全。
