# 注解

[[wiki/concepts/java-basic/注解与反射|返回注解与反射]]

## 这一层回答什么问题

> 注解到底是什么？元注解控制了什么？Spring 怎么"看到"@Component？

注解是代码的元数据——它贴标签但不做任何事。需要有人（反射、编译器、APT）去读它才有效果。理解元注解的 `@Retention(RUNTIME)` 是理解 Spring 扫描机制的前提。

## 注解的本质

```java
@Target(ElementType.METHOD)   // 能用在哪
@Retention(RetentionPolicy.RUNTIME)  // 保留到什么时候
public @interface LogExecution {
    String value() default "";
}
```

`@interface` 定义注解。`@LogExecution(value = "test")` 使用时 value 可以省略：`@LogExecution("test")`。

## 元注解

| 元注解 | 值 | 含义 |
|--------|-----|------|
| `@Target` | TYPE / METHOD / FIELD / PARAMETER... | 注解能贴在哪 |
| `@Retention` | SOURCE / CLASS / RUNTIME | 注解保留到什么时候 |
| `@Documented` | — | 是否出现在 Javadoc |
| `@Inherited` | — | 子类是否继承父类的注解 |

**`@Retention(RUNTIME)` 是 Spring 能工作的前提**——只有 RUNTIME 级别的注解，反射才能在运行时读到。SOURCE 级别的注解（如 `@Override`）编译后就丢了。

## 注解属性限制

属性类型只能是：基本类型、String、Class、枚举、注解，以及这些类型的数组。不能是自定义对象。

## Spring 怎么"看到"注解

```
1. @ComponentScan 找到目标包
2. 遍历所有 .class → Class.forName
3. isAnnotationPresent(Component.class) → 标记为候选 Bean
4. 读取 @Autowired 字段 → 查容器 → field.set
5. 读取 @Value("${...}") → 查配置文件 → 注入值
```
