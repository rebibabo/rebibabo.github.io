这一层回答什么问题

序列化是什么？Java 原生 Serializable 怎么用？为什么工程上都用 Jackson 而不是原生序列化？

序列化是把对象转成字节流（序列化）或从字节流恢复对象（反序列化）。日常开发中的典型场景：HTTP JSON 响应传给前端、Redis 存对象、RPC 远程调用传参。

Java 原生 Serializable

ObjectOutputStream.writeObject(obj) / ObjectInputStream.readObject() 完成序列化和反序列化。

缺点的代价：字节流体积大、不支持跨语言、serialVersionUID 管理不当会导致反序列化失败、transient 漏标注会泄漏敏感数据。工程上几乎不用原生序列化做网络传输。

Jackson JSON

Java 8 时间类型需要注册 JavaTimeModule，否则序列化出错。泛型反序列化用 TypeReference 绕过擦除：new TypeReference<List<User>>() {}。
