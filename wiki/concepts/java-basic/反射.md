# 反射

[[wiki/concepts/java-basic/注解与反射|返回注解与反射]]

## 这一层回答什么问题

> 反射是什么？怎么在运行时创建对象、调用方法、读写字段？为什么框架都在用反射？

正常写 Java 是编译期确定一切。反射反着来——运行时拿到一个类名字符串，就能创建对象、调用方法、读写字段。所有"框架魔法"的底层都是反射。

## 核心 API

```java
// 获取 Class 对象（三种方式）
Class<?> clazz = User.class;                           // 编译期已知
Class<?> clazz = obj.getClass();                       // 已有对象
Class<?> clazz = Class.forName("com.example.User");    // 运行时才知类名

// 创建对象
Constructor<?> ctor = clazz.getDeclaredConstructor(String.class);
Object obj = ctor.newInstance("张三");

// 读写字段（包括 private）
Field field = clazz.getDeclaredField("name");
field.setAccessible(true);      // 突破 private
field.set(obj, "李四");         // 写
String name = (String) field.get(obj);  // 读

// 调用方法
Method method = clazz.getDeclaredMethod("sayHello", String.class);
method.invoke(obj, "world");
```

## getXxx vs getDeclaredXxx

这个规律贯穿 Class、Constructor、Field、Method 四类 API：

- `getXxx()` → 拿 public 的（含从父类继承的）
- `getDeclaredXxx()` → 拿本类声明的（含 private，不含继承）

## 反射的性能

反射比直接调用慢 10-50 倍。原因：
- 需要做访问检查
- 参数需要装箱拆箱
- 方法内联困难

**缓存优化**：`getDeclaredMethod` 是重操作，拿到 Method 对象后缓存起来，重复用。

## 反射的应用

Spring 用反射做什么：
- 扫描 `@Component` → `Class.forName` + `isAnnotationPresent`
- DI 注入 → `field.set(target, bean)`
- 调 `@PostConstruct` → `method.invoke(bean)`
- 读 `@Value("${...}")` → `annotation.value()` + 反射
