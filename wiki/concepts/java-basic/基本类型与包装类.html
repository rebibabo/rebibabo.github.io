基本类型与包装类

[[wiki/concepts/java-basic/数据类型|返回数据类型]]

这一层回答什么问题

Java 的 8 种基本类型各占多少内存？为什么 Integer 和 int 能混用？== 比较包装类型为什么有时对有时错？

这是 Java 类型系统的地基。看似简单——但自动装箱的缓存池、拆箱的 NPE、浮点精度问题，是日常开发中最高频的坑。

8 种基本类型

类别 类型 大小 默认值 范围
整型 byte / short / int / long 1/2/4/8B 0 -2^(n-1) ~ 2^(n-1)-1
浮点 float / double 4/8B 0.0 IEEE 754
字符 char 2B（UTF-16 码元） 0~65535
布尔 boolean JVM 相关 false true/false

整型字面量默认 int，浮点默认 double。long a = 100L，float b = 1.0f。

类型转换

自动转换（Widening）：小范围 → 大范围，自动完成。int → long → float → double，不丢精度。表达式中有不同类型时，自动向更高精度提升。

强制转换（Narrowing）：大范围 → 小范围，需要显式 (type)。double pi = 3.14; int i = (int) pi; → 截断小数部分，不是四舍五入。

包装类型：为什么存在

基本类型不是对象——不能放进泛型集合（List<int> 不合法）、不能为 null、没有方法。包装类解决了这些问题。
