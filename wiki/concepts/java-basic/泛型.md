# 泛型

[[wiki/concepts/java-basic/Java-Basic-概念总图|返回概念总图]]

## 这一层回答什么问题

> 为什么 List 要写成 `List<String>`？类型擦除到底是什么？`? extends` 和 `? super` 什么时候用？

泛型是 Java 5 引入的类型参数化机制——让编译器在编译期帮你检查类型，而不是到运行时才抛 `ClassCastException`。

## 这层的主要分支

- [[wiki/concepts/java-basic/类型擦除|类型擦除]] — Java 泛型的核心实现机制，擦除的 5 个后果
- [[wiki/concepts/java-basic/通配符与PECS|通配符与 PECS]] — `? extends` / `? super` 的选择法则

## 没有泛型的世界

```java
List list = new ArrayList();
list.add("hello");
list.add(123);           // 编译不报错
String s = (String) list.get(1);  // 运行时 ClassCastException
```

有了泛型：

```java
List<String> list = new ArrayList<>();
list.add(123);           // 编译错误，根本过不去
String s = list.get(0);  // 不需要强转
```

## 泛型怎么用

```java
// 泛型类
public class Box<T> { private T content; }

// 泛型方法
public static <T> T first(List<T> list) { ... }

// 泛型接口
public interface Comparable<T> { int compareTo(T o); }
```

## 类型擦除：Java 泛型最核心的实现

Java 泛型是**编译期**的概念——编译完成后，所有类型参数被"擦除"成边界类型（默认 `Object`）：

```java
List<String> list1 = new ArrayList<>();
List<Integer> list2 = new ArrayList<>();
// 运行时 list1.getClass() == list2.getClass() → true
```

这是为了向后兼容——Java 5 的泛型代码要能在旧版 JVM 上跑。

**擦除的 5 个后果：**
1. 不能 `new T()` —— 运行时不知道 T 是什么
2. 不能 `List<int>` —— 基本类型不能作类型参数
3. 不能 `new T[10]` —— 泛型数组不合法
4. 不能 `instanceof List<String>` —— 运行时拿不到类型参数
5. 方法签名冲突 —— `void print(List<String>)` 和 `void print(List<Integer>)` 擦除后一样

## 通配符与 PECS

| 通配符 | 能读（get） | 能写（add） | 场景 |
|--------|------------|------------|------|
| `?` | 返回 Object | 几乎不能 | 不关心类型 |
| `? extends T` | 返回 T | 不能 | 只读数据源 |
| `? super T` | 返回 Object | 能写 T | 只写目的地 |

**PECS：Producer Extends, Consumer Super**

```java
// 从 src 读 → src 是 Producer → ? extends
// 往 dest 写 → dest 是 Consumer → ? super
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
    for (T item : src) dest.add(item);
}
```

记忆："我只读你，你 extend；我只写你，你 super"。

## 泛型和集合的关系

`ArrayList<String>` 和 `ArrayList<Integer>` 共用同一份字节码，但编译器保证你不把 Integer 放进只该有 String 的列表。集合框架之所以安全好用，全靠泛型这层编译期类型检查。
