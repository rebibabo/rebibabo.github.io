内存结构

[[wiki/concepts/java-basic/JVM|返回 JVM]]

这一层回答什么问题

一个 Java 对象到底存在哪？堆和栈有什么区别？什么情况下 OOM？

JVM 的内存不是一块大饼——它把内存分成几个功能不同的区域，每个区域有自己的职责、生命周期和 OOM 风险。

堆（Heap）

JVM 最大的内存区域，存放所有对象实例和数组。线程共享。GC 的主战场。

堆又分为：

  • 新生代：Eden + 两个 Survivor（S0、S1），默认比例 8:1:1。新对象在 Eden 创建，熬过一次 Minor GC 复制到 Survivor，熬过多次（默认 15 次）晋升到老年代
  • 老年代：长期存活对象、大对象（直接进入）

虚拟机栈（VM Stack）

线程私有。每个方法执行时创建一个栈帧，压入栈顶。栈帧包含：

  • 局部变量表：基本类型值、对象引用
  • 操作数栈：计算中间结果
  • 动态链接：指向运行时常量池的符号引用
  • 返回地址

递归过深或方法调用层次太多 → StackOverflowError。

方法区 / 元空间（Metaspace）

线程共享。存储类信息（字段、方法、构造方法）、运行时常量池、静态变量、JIT 编译后的代码缓存。

Java 8 以前叫永久代（PermGen），有固定大小上限，容易 OOM。Java 8+ 改为元空间，使用本地内存（Native Memory），默认不设上限——不再那么容易 OOM，但动态生成大量类（CGLIB、反射）仍可能撑爆。

程序计数器

线程私有。记录当前线程执行到的字节码行号——这是 JVM 里唯一不会 OOM 的区域。

各区域 OOM 总结

区域 异常 常见元凶
堆 Java heap space 对象太多、内存泄漏
栈 StackOverflowError 递归太深
元空间 Metaspace CGLIB / 反射动态生成大量类
直接内存 Direct buffer memory NIO 的 DirectByteBuffer 未释放

变量到底存在哪

  • 对象实例（new 出来的）→ 堆
  • 局部变量（方法里定义的）→ 虚拟机栈的局部变量表
  • 静态变量 → 方法区/元空间（Java 8+）
