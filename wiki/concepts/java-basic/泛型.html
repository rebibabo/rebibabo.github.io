类型擦除：Java 泛型最核心的实现

Java 泛型是编译期的概念——编译完成后，所有类型参数被”擦除”成边界类型（默认 Object）：

这是为了向后兼容——Java 5 的泛型代码要能在旧版 JVM 上跑。

擦除的 5 个后果：

  1. 不能 new T() —— 运行时不知道 T 是什么
  2. 不能 List<int> —— 基本类型不能作类型参数
  3. 不能 new T[10] —— 泛型数组不合法
  4. 不能 instanceof List<String> —— 运行时拿不到类型参数
  5. 方法签名冲突 —— void print(List<String>) 和 void print(List<Integer>) 擦除后一样

通配符与 PECS

通配符 能读（get） 能写（add） 场景
? 返回 Object 几乎不能 不关心类型
? extends T 返回 T 不能 只读数据源
? super T 返回 Object 能写 T 只写目的地

PECS：Producer Extends, Consumer Super

记忆：”我只读你，你 extend；我只写你，你 super”。

泛型和集合的关系

ArrayList<String> 和 ArrayList<Integer> 共用同一份字节码，但编译器保证你不把 Integer 放进只该有 String 的列表。集合框架之所以安全好用，全靠泛型这层编译期类型检查。
