Ad Server

[[wiki/series/programmatic-ads|返回程序化广告系列]]
[[wiki/concepts/programmatic-ads/角色总图|返回角色关系图]]

定义

Ad Server（广告服务器）是广告链路中的执行层。

如果说 [[DSP|DSP]]、[[SSP|SSP]]、[[ADX|ADX]] 更偏“决策和交易”，那 Ad Server 更偏：

  • 素材托管
  • 创意调度
  • 曝光点击记录
  • 跨媒体投放执行

两种常见视角

广告主侧 Ad Server

更像统一投放中心，帮助广告主跨媒体管理创意、曝光、点击和频控。

媒体方侧 Ad Server

更像广告位调度中心，负责媒体自有库存的展示、轮播、优先级和执行记录。

它解决什么问题

  • 创意素材不能散落在不同平台后台里
  • 展示、点击、转化需要统一记录
  • 频次控制和投放执行不能完全依赖单一交易平台

为什么它容易被忽略

很多讲程序化广告的材料会更多强调“谁在买、谁在卖、谁在撮合”，但真正把广告展示出来、把数据记下来，往往离不开 Ad Server 这种执行层。

所以它不是外围小工具，而是广告工程链路里的关键部件。

在系列里的位置

Ad Server 把程序化广告从“交易规则”延伸到了“广告真正如何被展示和记录”的工程实现层。
