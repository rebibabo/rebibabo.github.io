前言

写代码只是软件开发中的一个环节。入职后你会发现，真正的工作流程是：需求评审 → 技术方案 → 分支开发 → Code Review → 测试 → 部署 → 监控。这篇文章把一个需求从”产品经理提出”到”上线被用户使用”的完整链路走一遍，帮你在入职前建立全局视角。

6.3 发布策略

策略 做法 优点 风险
全量发布 所有实例同时更新 简单 出问题影响全部用户
灰度发布（推荐） 先更新 10% 的实例，观察正常后再全量 风险可控 需要基础设施支持
蓝绿部署 两套环境，切流量到新版本 秒级回滚 需要双倍资源
金丝雀发布 新版本只给少量用户（按用户 ID 或地区） 精细控制 实现复杂

10. 小结
